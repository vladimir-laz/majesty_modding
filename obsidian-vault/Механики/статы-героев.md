# Статы героев — где что лежит

Карта: какой параметр героя в каком файле менять. Сам герой описан в
[[def-сущности|<hero>.def]], собранном из экстендеров `extenders/*.inc`
(инлайнятся в `.def`, см. маркеры `;#line`). Пример разобран по
`mks_hero_minotaur.def` из [[4-кузница-у-монстров]].

## ⚠️ Базовые HP и урон — НЕ в этих файлах
В `.def` героя блок `extenders/health.inc` — **заглушка** (`value 1`, в
комментарии «начальное здоровье — не важно»). Реальные базовые HP, базовый
урон и рост характеристик по уровням берутся из **RPG-класса** (экстендер
`{"RPGModule"}` ссылается на профиль). Эти профили — в базовой игре, в
коллекции модов их нет. Поэтому «голую» силу героя по уровням здесь не
поменять — меняют то, что ниже.

## Что РЕАЛЬНО моддится
| Параметр | Файл | Где именно |
|---|---|---|
| **Цена найма** | [[shops-set]] | `{"<гильдия>" {units {"<hero>" {Cost {"count money" N}}}}}` |
| **Лимит героев** | [[shops-set]] | там же, `{"Max count" N}` |
| **Время найма** | [[shops-set]] | там же, `{"Production time" N}` |
| **Урон оружия / защита брони / бонусы характеристик** | [[unit-decour-xml]] | `<Weapon><Params>` (`f_dps_hth/range/magic`, `f_defence_*`, `f_strength`…) |
| **Доступный уровень снаряжения** | [[inventions-graphml]] + [[shops-set]] | узел изобретения + `{weapon}`/`{armour}` |
| **Стартовое оружие/броня** | `.def` | `extenders/inventory.inc` → `m2inventory` (`weapon_name`, `Armour type`) |
| **Способности/атаки** | `.def` | `extenders/actions.inc` → `ActionManager.actions` (можно раскомментировать) |
| **Разблокировка способности** | [[shops-set]] | `{"<гильдия>" {inventions {...}}}` |
| **Скорость передвижения** | `.def` | `extenders/mover.inc` → `{speed 3 4 4}`, `stepLength`, `angleSpeed` |
| **Радиус обзора** | `.def` | `extenders/sensor.inc` → `m2sensor {ViewRadius}` |
| **Снятие тумана** | `.def` | `extenders/fog_visor.inc` → `FogVisor {radius}` |
| **Макс. уровень** | `.def` | `extenders/level.inc` → `{MaxLevel 100}` |
| **Носимое золото / порог сытости** | `.def` | `extenders/unit.inc` → `TreasureKeeper "money" N`, `SatietyBarrier` |
| **«Успокоение» жителей** | `.def` | `extenders/safety_emitter.inc` → `{power} {radius}` |
| **Классы/теги** | `.def` | `extenders/entity_classes.inc` (`"hero"`, `"melee_hero"`, `"magical"`…) |
| **AI: приоритеты целей, что покупает** | `.def` | `extenders/ai.inc` → `HeroBehaviour` (`AllureFactors`, `PurchaseManager.TypeNecessity`) |
| **Имя/описание** | [[loctable-xml]] | ключи `#HERO_*_NAME` / `_DESC` |

## Практический вывод
- «Сделать героя сильнее в бою» проще всего через **оружие/броню** в
  [[unit-decour-xml]] (поднять `f_dps_*`, `f_defence_*`) — это и есть
  основной рычаг силы в Majesty.
- «Сделать дешевле/доступнее» → [[shops-set]] (цена, лимит, требования).
- «Дать новую способность» → раскомментировать/добавить в `actions.inc`
  + завести изобретение в `inventions` гильдии ([[shops-set]]).
