# Система спавна

## Суть
Существа и некоторые здания появляются на карте через **программы спавна**,
описанные в [[global-spawn-settings|global_spawn_settings.set]]. Здание
«запускает» программы через экстендер `spawner_extender` в своём
[[def-сущности|.def]]:

```
{"spawner_extender" {global_spawn_progs "progA progB"}}
```
`global_spawn_progs` — список имён программ через пробел; здание запускает
их все одновременно.

## Параметры программы
- `raw` — кого спавнить (id сущности).
- `count "min max"` — сколько за один спавн.
- `"spawn time"` — период (сек).
- `"max count"` — потолок одновременно живых **от этой программы**.
- `Nationality` — `Same` (свои) / `Enemy` (враги).
- `AI task` — поведение (бездействие `none`, агрессия, маршрут).

## Условный спавн
Программа может иметь `Prerequsities`. Виды условий из файлов:
- `building_level {Operation ">=" Level N}` — по уровню здания-владельца.
- `city_value {Min_city_summ Max_city_summ}` — по «ценности» города.
- `dead_heroes {Min_dead_heroes Max_dead_heroes}` — по числу павших героев
  (триггер кладбищ).
- `{"First wave"}` — флаг первой волны.

## Ключевые паттерны
- **Несколько программ = несколько источников.** Добавление второй
  программы в `global_spawn_progs` даёт дополнительный поток существ.
- **Условный клон для прогрессии:** второй спавнер = копия первого с
  `building_level >= 2`. Так башня на 2-м уровне получает второго лучника —
  см. [[второй-стражник]].
- **Логова** — здания, чей `spawner_extender` непрерывно порождает
  монстров; основа [[мини-сборка]] (см. [[логова-монстров]]).

## Где задаётся
- Программы: [[global-spawn-settings]].
- Запуск программ: `spawner_extender` в [[def-сущности]].
