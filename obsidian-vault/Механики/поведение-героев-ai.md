# Поведение и ИИ героев

В Majesty 2 героями нельзя командовать напрямую — они действуют сами по
ИИ. Сам **алгоритм принятия решений зашит в движок** и не редактируется;
моддер может лишь **калибровать коэффициенты (приоритеты)**, по которым
герой выбирает цель (подтверждено [[ссылки-и-источники|сообществом]]).

## Где настраивается
1. **Per-hero:** `entity/units/heroes/<hero>/extenders/ai.inc` →
   `Behaviour.HeroBehaviour` → `HeroTargetManager.AllureFactors`
   (раскрыто в [[def-сущности|.def]] героя).
2. **Глобально:** `set/ai_hero_allure_factors.set` — общие факторы/настройки
   по классам героев (там же флаг `{research N}`).

## AllureFactors (что на что тянет героя)
Коэффициенты «привлекательности» целей (пример из `mks_hero_minotaur`):
```
{AllureFactors
    {cl_dist 4}          ; тяга по расстоянию (ближе = привлекательнее)
    {cl_danger 1}        ; тяга к опасности
    {cl_flag_attack 2}   ; к флагу «атака»
    {cl_flag_protect 1}  ; к флагу «защита»
    {cl_flag_explore 0.5}; к флагу «разведка»
    {cl_theft 0}         ; к воровству
    {cl_building 1}      ; к зданиям
    {cl_research 1}      ; к исследованию territory/зданий
}
```
Также в `ai.inc`: `{crazy_factor N}`, `PurchaseManager.TypeNecessity`
(что герой покупает — armour/weapon/potion…), `relations`/`loot_relations`
(к кому относится дружелюбно/враждебно), `MotivationModifier`
(`HealthReflex {health_percent}` — когда бежит лечиться), `.sleep_in_guild`.

## `{cl_research}` — исследование карты (грабли)
По форуму: чтобы класс (рейнджер/разведчик) сам исследовал карту, правят
`{cl_research}` в его `ai.inc`. Значение **по умолчанию 2 — слишком мало**;
с ~5 рейнджеры начинают двигаться к разведке, 10 — заметнее. **Есть верхний
предел**: 200 эффекта не даёт (упирается в потолок). ⚠️ Опыт за такую
самостоятельную разведку герои НЕ получают.

## Рецепт: агрессивная оборона королевства
Чтобы герои сами шли бить монстров у владений — поднять **`cl_danger`** всем
героям (у воина/паладина/мага он крошечный: 0.05–0.1). В сборках выставлено
**1.5** всем 30 героям (у кого поле есть). Итоговая тяга к угрозе ≈ `cl_danger
× danger_1(30)` ≈ 45 — выше тяги к флагам (~4), но срабатывает ситуативно
(когда угроза в обзоре `m2sensor ViewRadius`). Подкрутка: 1.0 (мягче) … 2–3
(агрессивнее). Файлы: `entity/units/heroes/<hero>/<hero>.def`.

## ⚙️ ВАЖНО: глобальные МНОЖИТЕЛИ — `set/ai_hero_allure_factors.set`
Per-hero `cl_*` (в `.def`) **умножаются** на глобальные коэффициенты из
`set/ai_hero_allure_factors.set`:
```
{danger_1 30}{danger_2 20}     ; множитель опасности
{flag_attack 2}{flag_protect 1.5}{flag_explore 1}  ; множители флагов
{dist 0.1}{loot 1}{theft 1}...
```
**Итоговая тяга = `cl_X (деф) × X (глобал)`.** Пример:
- опасность: `cl_danger 1.5 × danger_1 30 = 45`
- флаг атаки: `cl_flag_attack 2.0 × flag_attack 2 = 4`

## ⚠️ ГРАБЛЯ: агрессия глушит реакцию на флаги
Подняв `cl_danger` 0.1→1.5 (агрессия), мы сделали тягу к опасности (45) в ~11×
больше тяги к флагам (4) → среди толпы монстров герои флаги ИГНОРЯТ. Фикс — не
трогая агрессию, поднять ГЛОБАЛЬНЫЕ множители флагов так, чтобы флаг победил
опасность: в сборках `flag_attack=flag_protect=flag_explore=30` → флаг
`2.0×30=60` > опасность `45`. Файл кладётся в `resource.mod/set/`.

## Рецепт: одинаковая реакция на все флаги
`cl_flag_attack/protect/explore` (деф) — уравнены = **2.0** всем 30 героям; плюс
ГЛОБАЛЬНЫЕ множители флагов уравнены = **30** (см. выше). Итог: герои одинаково
охотно идут на любой флаг, и флаг перебивает агрессию.

## Рецепт: индивидуальный порог отступления (эльфийка 35%)
`HealthReflex {health_percent N}` можно задавать ПОФАЙЛОВО. Базово у всех 20%
(см. ниже), но конкретному классу можно поднять/опустить. Сделано: `hero_elf`
(эльфийка) — **35%** (отступает раньше, она хрупкая). Менять только в её `.def`.

## Рецепт: убрать трусость (бегство от опасности), оставив отступление по HP
В `ai.inc` героя (раскрыт в `.def`) блок `{"MotivationModifier" {MotivationModifiers ...}}`
содержит **два рефлекса**:
```
{HealthReflex {health_percent 20}}   ; уйти лечиться при HP < 20% — отступление по ХП
{SafetyReflex {}}                    ; бегство от ОПАСНОСТИ (страх) — это «трусость»
```
- **`SafetyReflex`** (всегда пустой `{}`, параметров нет) = бегство, когда рядом
  опасность, которую герой считает чрезмерной. Усиливается при росте числа
  монстров. Тюнить нельзя — только **удалить** блок → герой перестаёт убегать
  «от страха», дерётся по AllureFactors и отступает только по HP.
- **`HealthReflex {health_percent N}`** — порог отхода на лечение. Разброс в базе
  5–50% (50 = бежит рано/трусит, 5 = почти до смерти). Унифицировать.

**Сделано в сборках:** у всех 30 героев (с фреймворком рефлексов) `SafetyReflex`
убран, `HealthReflex` = **20%** всем. ⚠️ `health_percent` бывает ДРОБНЫМ (`10.0`) —
regex `\d+` пропустит, брать `[\d.]+` (та же грабля, что с HP зданий).
4 героя без фреймворка (`hero_ice_mage`/`death_knight` — стаб-деф без
`MotivationModifier`; `paladin_spirit`/`boss_paladin_spirit` — призываемые духи,
не нанимаются) — **намеренно не трогали** (остаются «до смерти»).

## Что НЕ моддится
Сама логика «как герой решает» — в движке. Нельзя написать новый алгоритм
поведения, только подкрутить веса. Командовать героем напрямую нельзя
(дизайн игры).
