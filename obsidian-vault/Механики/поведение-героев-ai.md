# Поведение и ИИ героев

В Majesty 2 героями нельзя командовать напрямую — они действуют сами по
ИИ. Сам **алгоритм принятия решений зашит в движок** и не редактируется;
моддер может лишь **калибровать коэффициенты (приоритеты)**, по которым
герой выбирает цель (подтверждено [[ссылки-и-источники|сообществом]]).

## Где настраивается
1. **Per-hero:** `entity/units/heroes/<hero>/extenders/ai.inc` →
   `Behaviour.HeroBehaviour` → `HeroTargetManager.AllureFactors`
   (раскрыто в [[def-сущности|.def]] героя).
2. **Глобально:** `set/ai_hero_allure_factors.set` — общие факторы/настройки
   по классам героев (там же флаг `{research N}`).

## AllureFactors (что на что тянет героя)
Коэффициенты «привлекательности» целей (пример из `mks_hero_minotaur`):
```
{AllureFactors
    {cl_dist 4}          ; тяга по расстоянию (ближе = привлекательнее)
    {cl_danger 1}        ; тяга к опасности
    {cl_flag_attack 2}   ; к флагу «атака»
    {cl_flag_protect 1}  ; к флагу «защита»
    {cl_flag_explore 0.5}; к флагу «разведка»
    {cl_theft 0}         ; к воровству
    {cl_building 1}      ; к зданиям
    {cl_research 1}      ; к исследованию territory/зданий
}
```
Также в `ai.inc`: `{crazy_factor N}`, `PurchaseManager.TypeNecessity`
(что герой покупает — armour/weapon/potion…), `relations`/`loot_relations`
(к кому относится дружелюбно/враждебно), `MotivationModifier`
(`HealthReflex {health_percent}` — когда бежит лечиться), `.sleep_in_guild`.

## `{cl_research}` — исследование карты (грабли)
По форуму: чтобы класс (рейнджер/разведчик) сам исследовал карту, правят
`{cl_research}` в его `ai.inc`. Значение **по умолчанию 2 — слишком мало**;
с ~5 рейнджеры начинают двигаться к разведке, 10 — заметнее. **Есть верхний
предел**: 200 эффекта не даёт (упирается в потолок). ⚠️ Опыт за такую
самостоятельную разведку герои НЕ получают.

## Рецепт: агрессивная оборона королевства
Чтобы герои сами шли бить монстров у владений — поднять **`cl_danger`** всем
героям (у воина/паладина/мага он крошечный: 0.05–0.1). В сборках выставлено
**1.5** всем 30 героям (у кого поле есть). Итоговая тяга к угрозе ≈ `cl_danger
× danger_1(30)` ≈ 45 — выше тяги к флагам (~4), но срабатывает ситуативно
(когда угроза в обзоре `m2sensor ViewRadius`). Подкрутка: 1.0 (мягче) … 2–3
(агрессивнее). Файлы: `entity/units/heroes/<hero>/<hero>.def`.

## Что НЕ моддится
Сама логика «как герой решает» — в движке. Нельзя написать новый алгоритм
поведения, только подкрутить веса. Командовать героем напрямую нельзя
(дизайн игры).
