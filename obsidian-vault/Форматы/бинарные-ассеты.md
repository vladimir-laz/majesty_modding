# Бинарные ассеты — модели, анимации, текстуры

## Описание
Часть файлов мода — бинарные ресурсы, которые **не редактируются текстом**
(только через спец-инструменты или замену готовых файлов). В коллекции они
встречаются в [[мини-сборка]] (модели монстров/логов) и
[[4-кузница-у-монстров]] (модели оружия/брони/зданий).

## Типы
| Расширение | Что это |
|---|---|
| `.mdl` | 3D-модель сущности (привязывается к `.def` через блок `structure`) |
| `.mesh` | геометрия (меш) модели |
| `.material` / `.mtl` | материалы/шейдинг модели |
| `.anm` | анимация (`walk.anm`, `blow01.anm`, `death.anm`, `idle.anm`, `stun.anm`) |
| `.dds` | текстура (DirectDraw Surface), в т.ч. атласы кнопок UI |
| `.tuv` | UV-координаты для атласа `.dds` (см. [[texture-units-xml]]) |

## Сопутствующие текстовые файлы
Рядом с бинарниками юнита лежат текстовые описания (их можно читать):
- `anim.txt`, `anim.inc` — список и склейки анимаций.
- `fsm.inc`, `fsm.graphml` — конечный автомат (FSM) поведения юнита.
- `.def` / `.mdl` бывают и текстовыми описателями того, как собрать модель.

## Практика
Моддеры обычно **переиспользуют** существующие модели/анимации
(ссылаются на ассеты базовой игры из `.def`) и добавляют свои `.dds` лишь
для иконок новых кнопок. Полноценные новые модели — редкость.
