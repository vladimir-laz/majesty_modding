# Формат `.set` / `.def` — вложенные фигурные скобки

## Описание
Базовый текстовый формат данных Majesty 2. Используется в `.set`
(конфиги), `.def` (сущности) и `.inc` (включаемые куски). Синтаксис —
дерево узлов во вложенных фигурных скобках, похоже на S-выражения.

## Синтаксис
```
{"ИмяУзла"
    {ключ значение}
    {ключ "значение со пробелами"}
    {ВложенныйУзел
        {подключ 123}
    }
}
```

Правила:
- **Узел** = `{ ... }`. Первый токен часто имя/тип, остальное — содержимое.
- **Имена с пробелами** берутся в кавычки: `{"count money" 550}`,
  `{"Max count" 3}`, `{"spawn time" 15}`.
- **Значение-пара**: `{ключ значение}` — например `{money}` (флаг без
  значения), `{Cost 35.0}`, `{count "1 1"}` (строка из двух чисел).
- **Комментарий** — точка с запятой `;` до конца строки.
- **Числа** — целые и с плавающей точкой (`0.2`, `35.0`).
- **Флаги** — узел без значения: `{postspawn}`, `{has_static}`, `{money}`.

## `{version ...}`
Очень частый служебный узел:
- `{version original}` — запись наследует значения из оригинала игры (в
  `shops.set` так помечен почти каждый блок здания).
- `{"building" {version 26}}` — номер версии формата сущности в `.def`.
Версию обычно **не трогают**, если не меняют структуру записи.

## Кодировка и «кракозябры»
Файлы часто в кодировке Windows-1251. Кириллические комментарии в `.def`
(например `;	����� �������� ������`) в UTF-8 выглядят битыми — это
нормально, на работу не влияет, это просто комментарии разработчиков.
