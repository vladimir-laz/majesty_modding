# Установка и совместимость модов

## Установка (из `Порядок установки.txt`)
Во всех модах коллекции инструкция идентична:

- **Установка:** переместить папку `resource.mod` из архива мода в папку с
  игрой.
- **Удаление:** удалить папку `resource.mod` из папки с игрой.

То есть установка = простое копирование папки, никакого инсталлятора. Это
прямое следствие механизма переопределения файлов (см. [[система-модов]]).

## Почему моды конфликтуют
Два мода **конфликтуют, если переопределяют один и тот же файл** — при
копировании второй мод предложит заменить файл первого, и в игре останется
только одна версия. Примеры пересечений в коллекции:

- `set/trading/shops.set` правят сразу **4 мода** ([[рыцарь]],
  [[элитные-башни]], [[4-кузница-у-монстров]], [[мини-сборка]]) → они
  попарно конфликтуют по этому файлу.
- `gameData/spawn/global_spawn_settings.set` правят [[второй-стражник]] и
  [[мини-сборка]].
- `interface/dynamic/menu/castle_menu.xml` — [[элитные-башни]] и
  [[мини-сборка]].

## Правило совместимости
Из инструкции: **моды, которые при установке НЕ предлагают замену файлов,
совместимы между собой.** Иначе говоря: если наборы файлов двух модов не
пересекаются — их можно ставить вместе.

Полностью совместимы со всеми, т.к. трогают уникальные файлы:
- [[10x-скорость]] — только `interface/.../game_window.xml`.
- [[второй-стражник]] частично: делит `global_spawn_settings.set` с
  [[мини-сборка]], но не делит его с остальными.

Чтобы объединить конфликтующие моды, нужно **вручную слить** их правки в
один общий файл (например, один `shops.set`, содержащий изменения и
рыцаря, и элитных башен).
