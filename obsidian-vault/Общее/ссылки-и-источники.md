# Внешние источники, инструменты и грабли моддинга M2

Сводка знаний из сообществ Majesty 2 (то, чего не вывести из самих файлов
модов). Источники — внизу.

## `resource.pak` и проверка целостности
- Контент игры лежит в **`resource.pak`** (+ `resource.expansion_*`).
- Игра проверяет целостность `.pak`: **размер кратен 16 байт** и **контрольная
  сумма совпадает с зашитой в `.exe`**. Прямая правка `.pak` → ошибка
  *«Application aborted! … data integrity was damaged, changes in resources
  detected»*.
- **Поэтому моддят через `resource.mod`** — он переопределяет `.pak` и
  обходит проверку (см. [[система-модов]]). Файл кладут по ПОЛНОМУ пути
  (`resource.mod/set/trading/shops.set`, не в корень) — иначе правка не
  применяется.

## Инструменты
- Распаковка `resource.pak` — обычные распаковщики (**7-Zip** работает,
  Total Commander). Некоторые `.pak` зашифрованы — архиваторы-распаковщики
  справляются лучше менеджеров.
- **Редактор GEM** (`editor [GEM]`) — родной редактор движка. Грузит ВСЕ
  ресурсы сразу (жадно), поэтому строго реагирует на отсутствующие файлы —
  см. грабли ниже и [[ссылки-и-источники]].

## Карта файлов
Полная карта расположения конфигов — [[карта-игровых-файлов]] (там же
`spells.xml`, `rpg_params.xml`, `caravans.set`, `potions.set` и др.).

## Известные грабли
- **Правка не действует** → файл лежит не по полному пути в `resource.mod`.
- **Краш `File not found … .tuv/.dds` (`repository.cpp`)** → `textures.xml`
  объявляет атлас, файла которого нет (частая беда при сборках). Чинится
  перенаправлением атласа на существующий или подкладыванием файла — см.
  разбор в нашем кейсе. Редактор GEM ловит это раньше игры (жадная загрузка).
- **Пропадают способности юнита после правки `entity/`** — известная
  проблема сообщества без готового решения.
- Герои растут до **уровня 100**, но опыт по уровням — экспоненциально
  (практический максимум ~35+).
