# Рецепт: увеличить спавн монстров из логов

## ⚠️ Важная оговорка (исправление)
Глобальный файл ниже влияет на спавн **логов, построенных ИГРОКОМ**, и на
миссии без своей коллекции спавна. НО **кампанийные лога берут спавн из
самой миссии** (`.mi`, `helper/spawn_program_collection`), а не отсюда —
для них см. [[спавн-и-деньги-в-миссиях]].

## Где (глобально)
`gameData/spawn/global_spawn_settings.set` — глобальный файл (см.
[[система-спавна]], [[global-spawn-settings]]).
У каждого монстра — программа спавна:
```
{"beast_wolf"
    {count "1 1"}      ; сколько за один спавн (min max)
    {"spawn time" 15}  ; период, сек
    {"max count" 2}    ; потолок одновременно живых
}
```

## Какие программы править (только логовые)
Не все 175 программ — логова. Нужные = те, на которые ссылаются
`logovo_*.def` в `spawner_extender.global_spawn_progs`. Собрать список:
```
grep -rh global_spawn_progs entity/buildings/logovo_*/  → имена программ
```
(в сборке `more_monsters` так найдено 43 программы).

## Рычаги
- `count "min max"` ↑ — больше за один спавн.
- `"max count"` ↑ — больше монстров одновременно (главный рычаг «их много»).
- `"spawn time"` ↓ — чаще (но кулдаун).

Пример (сборка `more_monsters`): всем логовым программам `count` +2 (1→3) и
`max count` ×2 (2→4), `spawn time` без изменений.

## ⚠️ Важно
- Это НЕ про флаг `randomizable` (тот лишь рандомизирует ПОЗИЦИИ логов,
  не количество — патч 1.3).
- Per-логово параметра количества нет: логово лишь ссылается на общую
  программу. Поэтому изменение одинаково для всех логов данного типа.
- Программы общие для логов игрока и ВРАЖЕСКИХ → врагов тоже станет больше.
- «Слабо в ранних миссиях, много в поздних» — это число/тип/уровень логов,
  заданные в самих миссиях (.mi), глобально не выровнять.
