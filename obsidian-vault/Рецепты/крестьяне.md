# Рецепт: крестьяне — число в домах и их HP

## Число крестьян в доме → `global_spawn_settings.set`
Крестьян из дома плодит программа **`peasant_spawner`** (глобальная):
```
{"peasant_spawner"
    {Prerequsities {building_level {Operation "=="}{Level 1}}}
    {raw peasant_female}
    {count "1 1"} {"spawn time" 10} {"max count" 2}   ; 2 крестьянина на дом
    {"First wave"}
}
```
- `"max count"` = сколько крестьян живёт в доме (база 2). Поднять → больше
  крестьян (и больше налогов). В сборках поднято до **4**.
- Программы `peasant_house_spawner1/2/4` — это спавн самих ДОМОВ (расширение
  города, `spawn type buildings`), не путать с населением дома.

## HP крестьян → `rpg_params.xml` (РАЗБЛОКИРОВАНО)
HP крестьянина (база **30**) — в [[rpg-params-xml]], секция `<Citizens>`
(`peasant_female`/`peasant_male`, `f_maxHealth`). В `.def` крестьянина
Health — заглушка (`value 1`). Раньше блокировалось неполной копией, теперь
есть полный `rpg_params.xml` (`majesty_rpg/`) → в сборках HP крестьян **×10 = 300**.

## ♻️ Вечный реген крестьянам (как от амулета регенерации)
Навесить ПЕРМАНЕНТНЫЙ перк-реген на крестьянина. Перки-регены — в
[[perks-xml|perks.xml]]: блок `HealthChange` + `f_value` за `f_period`; БЕЗ
`<f_liveTime>` = постоянный. Готовый «амулетный» (умеренный):
```
artefact_of_regeneration : f_period 1, HealthChange f_value 2   ; +2 HP/сек, вечный
```
(Боссовские `state_werewolf_regeneration`=+350, `ogre_regeneration` — слишком
сильные, не брать.) Навешивается через `RPGModule` в **дефе** крестьянина:
```
{"RPGModule"
    {perks
        {perkinstance "artefact_of_regeneration"}
    }
}
```
У базового `peasant_male/female.def` блок `{"RPGModule"}` пустой — заменить на
вид выше. Перк берётся из базового `perks.xml` (отдельно класть не нужно).
Сделано в сборках для `peasant_male` и `peasant_female` (оверрайд `.def`).
> Механизм универсален: так можно дать вечный пассив (реген/иммунитет/защиту)
> ЛЮБОМУ юниту — см. `default_perks.inc` у героев/боссов (`perkinstance`).
