# Рецепт: добавить монстрское здание (mks_*) в человеческий замок

Зачем: чтобы за людей можно было строить монстрские здания — гильдию
оборотней (`mks_guild_werewolf`) и кузницу монстров (`mks_smithy`, нужна для
экипировки нанятых оборотней/монстро-героев). Сделано в текущих сборках.

## Что нужно (2 файла)
Сущность и текстура кнопки берутся из БАЗОВОЙ игры (mks-контент) — их
поставлять не надо; в моде только «крючки»:

### 1. `set/trading/shops.set` — сделать здание строящимся людьми
В блок **человеческого** замка `{"castle" {buildings { ... }}}` (это до
`{"mks_castle"}`!) добавить запись здания с ЧЕЛОВЕЧЕСКИМИ требованиями:
```
{"mks_smithy"
    {version original}
    {Cost {money} {"count money" 500}}
    {Entity mks_smithy}
    {"Max count" 100}
    {"Max upgrade" 3}
    {Prerequesities {gold 0} {"heroes count" 0}}
    {"Upgrades prerequsity" {"1" {buildings castle_2}} {"2" {buildings castle_3}}}
}
```
⚠️ В оригинале у `mks_*` пререквизиты ссылаются на `mks_castle_2` и т.п. —
заменить на человеческие (`castle_2`/`castle_3`), иначе людям не построить.

### 2. `interface/dynamic/menu/castle_menu.xml` — кнопка в меню замка
В нужную группу (`castle_build_std` для кузницы, `castle_build_guild` для
гильдии) добавить кнопку (текстура `BUT_build_<entity>` уже есть в базе —
её юзает `mks_castle_menu.xml`):
```xml
<Button_4 action="CastleMenuItemClickAction" stateAction="CheckBuildAvailableAction" transition="castle_build_in_progress">
  <appearance texture="BUT_build_mks_smithy" autoTexture="TEX_MASK" autoHint="false" propertyBlock="action_but" userdata="mks_smithy"/>
</Button_4>
```
`userdata` = id сущности; `transition="castle_build_in_progress"`.

## Проверка связей
кнопка(`userdata`) → запись в `castle.buildings`(`Entity`) → сущность из базы;
текстура `BUT_build_*` → база (та же, что в `mks_castle_menu`).

## Сделано в сборках
- `mks_guild_werewolf` (гильдия оборотней) — группа гильдий, req `castle_2`.
- `mks_smithy` (кузница монстров) — группа «Постройки», req `castle_2`/`_3`.

## Апгрейд монстрского здания без гейта по замку
 — общая сущность для людей и монстров, разный гейт по фракции
невозможен. Решение: УБРАТЬ блок  из записи здания —
тогда апгрейд без требования замка (как у /, у которых
этого блока нет). "Нет блока = нет гейта по замку" (движок НЕ дефолтит на замок).
