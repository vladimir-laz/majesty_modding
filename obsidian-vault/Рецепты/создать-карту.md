# Рецепт: создать новую карту/миссию

## Из чего состоит карта (папка миссии в `resource/map/.../<mission>/`)
| Файл | Что | Редактируется руками? |
|---|---|---|
| `.heightmap` (~2.5 МБ) | рельеф (высоты) | ❌ бинарный |
| `.mesh` (~2 МБ) | геометрия террейна | ❌ бинарный |
| `.grass` | расстановка травы | ❌ бинарный |
| `.material` | материалы террейна | ❌ бинарный |
| `map` (~140 КБ) | расстановка ОБЪЕКТОВ и ЗОН (замки, логова, зоны спавна, теги) | ❌ бинарный |
| `<mission>.mi` | ЛОГИКА: королевства, деньги, спавн, цели, доступные здания | ✅ текст ([[mission-mi]]) |
| `.info` | `{name ""}{description ""}` | ✅ текст |
| `minimap.dds` / `mission_image.dds` | миникарта/превью | картинка |

Одиночные/MP миссии (`Single_missions/`, `Multiplayer_missions/`) часто содержат
только `map` + `minimap` + `.mi` (террейн внутри `map`). Кампании регистрируются
в `resource/map/<Campaign>/campaign.xml` (`<Level>` → `<s_trueFolder>` = папка
миссии, позиция на карте кампании, кнопки, условия открытия).

## ⭐ Новую карта С НУЛЯ (новый рельеф) → ТОЛЬКО официальный редактор
Рельеф/объекты/зоны — бинарные (`map`/`.heightmap`/`.mesh`/`.grass`), их
**нельзя написать руками** — только сгенерировать **официальным редактором**
`M2Editor.exe`:
- лежит в корне игры (`...\Steam\steamapps\common\Majesty 2 Collection\M2Editor.exe`);
- или Steam → ПКМ по «Majesty 2 Collection» → пункт **map editor**; лаунчер
  тоже спрашивает «editor или игра».
- Горячие клавиши: **F2** = редактор карты (рельеф/объекты), **F3** = редактор
  миссии (квесты/скрипты), **F6** = свет/погода, **F7** = теги сущностей,
  **F9** = королевства, `Shift+Alt+T` = торговые правила, `Shift+Alt+K` =
  создать королевство, `Shift+Alt+D` = дипломатия.
- ⚠️ Редактор капризный: проще **открыть существующую миссию** и править её,
  чем начинать с пустого листа (у пустого многое в свойствах миссии недоступно).

## Что МОЖНО без редактора (клон + новый сценарий)
Сделать новую ИГРУ на СУЩЕСТВУЮЩЕМ рельефе:
1. Скопировать папку готовой миссии под новым именем.
2. Переписать `.mi` ([[mission-mi]]): королевства/деньги, спавн логов, цели,
   доступные здания. Использовать ТОЛЬКО уже размещённые в `map` зоны/теги
   (новые логова/замки в новых координатах = нужен редактор).
3. Прописать `.info` (имя/описание) и зарегистрировать (campaign.xml как
   `<Level>` либо как одиночную миссию).
Ограничение: ландшафт и расположение объектов остаются от исходной карты.

## Условия миссии (для своих сценариев) — в `.mi` → `helper/quests`
Каждый квест: `launch_condition` / `victory_condition` / `defeat_condition` +
`*_reaction`. Доступные условия (по всем миссиям игры):
- **`Annihilation`** {Tag ...} — уничтожить цель (победа: снести все логова;
  поражение: `{Tag player_castle}` разрушен — стандартная «оборона»).
- **`Counter`** — счётчик (через триггеры; так делают «выжить N дней», текст `#QuestCounterDays`).
- **`Building exist`/`Building upgrade`**, **`Check level`** (герой N ур.),
  **`Check player gold`**, **`Arrival`** (приход в зону), **`Tags count`**,
  **`and`/`or`/`not`** (комбинаторы).
Деньги/заклинания игрока — в `{"kingdoms"}` (`kingdomtreasure "money" N`, `spellbook`).

## ✅ Сделано: миссия-оборона «Последний оплот» (клон PP04)
Одиночные миссии **сканируются из папки** `map/Single_missions/` (списка-конфига
нет; имя/описание — из `.info`, кодировка cp1251). Значит новая папка появляется
в меню сама. Рецепт, который сработал:
1. Скопировать папку `Single_missions/PP04/` → `Single_missions/last_bastion/`.
   ⚠️ **ГРАБЛЯ:** файлы рельефа **СКРЫТЫЕ** (с точки): `.mesh`, `.heightmap`,
   `.grass`, `.material` — обычный `cp file*` и `ls` их пропускают! Без `.mesh`
   игра падает: `File not found - /map/.../<msn>/.mesh (repository.cpp)`.
   Копировать ВСЮ папку (`cp -a`/`cp .*`). Видимые: `map`, `minimap.dds`,
   `mission_image.dds`, `<msn>.mi`. (`.occlusion` опционален — у PP04 его нет.)
2. `.info` (cp1251): `{name "Последний оплот"}{description "..."}`.
3. В `.mi`: деньги игрока (`{humanplayer 1}` → `kingdomtreasure "money" 50000`),
   рескин страшных монстров (`{raw undead_zombie}`→`{raw beast_werewolf}`,
   archer_skeleton→`fantasy_elemental_fire`, bear→`fantasy_elemental_earth`),
   буст спавнов (none/protect: count "4 5", max 10, st 18; attack_in_zone: count
   "3 3", st≤30, max≥8). Парсер коллекции: искать `spawn_program_collection`
   БЕЗ кавычек (в файле `"helper/spawn_program_collection"`).
4. Квест PP04 переиспользован как есть: победа = `Annihilation {Tag mage_castle}`
   (снести вражеский оплот), поражение = `Annihilation {Tag player_castle}`.
**Ограничения v1:** ландшафт и расположение замка/логов — от PP04 (свой угол/
рельеф = только редактор). «Уничтожить ВСЕ логова» одним условием нельзя —
`Annihilation` целится по ОДНОМУ тегу, а у логов общего тега нет (нужно тегать
каждое логово в блоках `{entity "logovo_*" 0x.. ...}` — инвазивно). Альтернатива
победы «выжить N дней» = квест с `Counter` + триггер инкремента (как pp15/19).

## ⚠️ Удаление здания/логова из `.mi` — НЕ просто `{entity}`!
Размещённое здание = МНОГО связанных структур. Удалив только `{entity "logovo_X" 0xID}`,
получишь краш **`Building piece ...: missing owner building! (BuildingPiece.cpp)`**.
Чтобы убрать здание с id `0xID` ЧИСТО, удалить ВСЁ:
1. главную `{entity "logovo_X" 0xID ...}` + `{entity "logovo_X_l0" ...}`;
2. сущности-части `{entity "object.._ground_logovo_X" 0xPID}` (их id берутся из links);
3. `{extenders 0xPID {"buildingpiece" {ownerbuilding 0xID}...}}` — связки часть→владелец;
4. `{extenders 0xID {"home" ...}}` — расширитель самого здания;
5. `{link 0xPID {0xID "dummyNN"}}` — все link-связи на 0xID;
6. `{tags "имя" 0xID}` — метки здания.
**Проверка:** ни одного вхождения `0xID` как точного id (граница — не hex-символ);
все ссылки (ownerbuilding/extenders/tags/link) резолвятся в существующие `{entity}`;
баланс `{}`. Вывод: чистое удаление зданий — реально, но кропотливо; в редакторе проще.

## Перенос базы игрока (замка) без редактора
1. Поставить замок на место СУЩЕСТВУЮЩЕЙ постройки (там валидная земля) — взять её позицию.
2. Вектор V = (поз_постройки − поз_замка). Сдвинуть на V ВСЕ {position X Y} в боксе базы (замок+его части l0/l1/l2/торчи/bg, дома, рынок, кузница + их части). Сущности игрока — nation player (kingdomdescriptor "player").
3. ⚠️ Зоны в helper/zones заданы координатами (circle {position}{radius}). Зону around_player_castle (куда летят волны attack_in_zone) сдвинуть ТЕМ ЖЕ V, иначе монстры бьют пустое место.
4. Проверить, что бокс не захватил чужие сущности/логова (враг nation enemy обычно далеко).

## Победа «уничтожить все логова/здания X» + нейтралитет (сделано в last_bastion)
- **Цель «снести все X»:** пометить все нужные сущности ОБЩИМ тегом — по одной
  строке `{tags "all_lairs" 0xID}` на здание в секции тегов (рядом с `{link ...}`),
  и условие победы:
  ```
  {"Tags count" {properties {Tag all_lairs}{Operation "=="}{Value 0}}}
  ```
  (= победа, когда сущностей с тегом 0). Так сделано в кампании (`red_castle` и т.п.).
  В last_bastion помечено 24 логова, победа = снести все.
- **Нейтралитет фракции:** блок `{"diplomacy"}` → `{relations}` = список
  `{diplomacyrelation N}` по парам королевств. Порядок пар: idx0=player-enemy,
  idx1=player-creatures, … последний = creatures-rats. **1=враг, 0=нейтрал, 2=союз.**
  Сменить нужную пару на 0. У нас: player-enemy(маги) `1→0` (нейтрал),
  player-creatures(монстры) остался `1` (враг → их и зачищаешь).
  Отключённые геройские спавны вернуть (`count "0 0"`→`"1 1"`, `max 0`→`2`) —
  нейтралы не нападают, просто живут у своей базы.
